## 磁盘逻辑结构图 (FAT12, 1.44MB 软盘)

```mermaid
block-beta
  columns 1
  block:Sectors
    columns 18
    S1["S1\n引导扇区\n(0x7C00)"] 
    S2["S2\nFAT#1"] 
    S3["S3"] 
    S4["S4"] 
    S5["S5"] 
    S6["S6"] 
    S7["S7"] 
    S8["S8"] 
    S9["S9"] 
    S10["S10\nFAT#1 结束"] 
    S11["S11\nFAT#2 开始"] 
    S12["S12"] 
    S13["S13"] 
    S14["S14"] 
    S15["S15"] 
    S16["S16"] 
    S17["S17"] 
    S18["S18"] 
  end
  S1Note["BIOS 自动加载到 0x7C00"]  
  S2to18["S2~S18 由代码加载到 0x8200\n(共17个扇区, 0x8200~0xA3FF)"]  
```

> **说明：**  
>
> - **S1 (扇区1)**：引导扇区，BIOS 自动载入物理地址 `0x7C00`。  
> - **S2 ~ S18 (扇区2~18)**：你的代码从柱面0、磁头0、扇区2开始，循环读取，直到扇区18，全部装入 `0x8200` 开始的内存。  
> - 按照标准的 FAT12 结构，S1~S18 包括：引导扇区、FAT1 (9个扇区)、FAT2 (9个扇区)，以及根目录区的起始部分（通常紧随 FAT2 之后）。你的加载范围正好覆盖了这些关键区域。

## 内存分布图

```mermaid
flowchart LR
    subgraph Physical Memory
        direction TB
        A["0x00000"]
        B["0x07C00\n---- 引导扇区 (512B) ----\n由 BIOS 自动加载"]
        C["0x07E00"]
        D["0x08200\n---- 加载的扇区 S2~S18 (8704B) ----\n代码循环读取"]
        E["0x0A400"]
        F["更高地址..."]
    end
    A --> B --> C --> D --> E --> F
```

> **要点：**  
>
> - 两个区域之间有空隙 `0x7E00 ~ 0x8200`（正好 1KB 的间隔），这是有意设计的，避免堆栈和数据互相覆盖（堆栈在 `SS:SP = 0x0000:0x7C00`，向下增长，不会侵入数据区）。  
> - 读取的 17 个扇区包含了 FAT 表和根目录，后续可以在这些数据里解析文件。

这样图文结合，磁盘读取的逻辑就一目了然了。
