## rozi00d.nasm 深度解析与知识串联

### 一、文件概览

这个 NASM 程序是一个 **FAT12 引导扇区**，它会把自己加载到 `0x7C00` 后，再读取软盘的绝大部分数据到内存中，最后打印一段信息。相比 `rozi00c.nasm`，它新增了多柱面、多磁头的读取能力。

### 二、核心知识点回顾

#### 1. 实模式分段寻址 —— 一切的基础

```
物理地址 = 段寄存器 × 16 + 偏移地址
```

- **段寄存器**（CS, DS, ES, SS）值每加 1，物理地址移动 16 字节。
- **偏移地址**（SP, BX, SI 等）直接按字节寻址。
- 段寄存器赋值时必须先除以 16：
  - `0x7C00` → 堆栈偏移（`SP` 可直接赋值，因为 `SS=0`）
  - `0x8200` → 数据段基址 `ES = 0x0820`（`0x8200 / 16`）

**为什么是 ×16？**  
8086 用 16 位寄存器生成 20 位地址：硬件将段寄存器的 16 根信号线接到加法器的第 4～19 位，低 4 位接地，相当于左移 4 位（×16），无运算、无数据丢失。

#### 2. 磁盘读取逻辑（INT 0x13）

- **CH** = 柱面号（0～79 或更多）
- **DH** = 磁头号（0 或 1）
- **CL** = 扇区号（1～18）
- **ES:BX** = 数据加载的目标内存地址
- `AH=0x02` 读盘，`AL=扇区数`，`DL=驱动器号`
- 失败时 `CF=1`，可重置磁盘（`AH=0x00`）后重试

#### 3. 循环结构：从单柱面到全盘

`rozi00d.nasm` 使用了 **三层嵌套循环**：

```mermaid
flowchart TD
    A[开始 C0, H0, S2] --> B[读取当前磁头下的扇区]
    B --> C{扇区号 <= 18 ?}
    C -->|是| B
    C -->|否| D[CL=1, DH+1]
    D --> E{DH < 2 ?}
    E -->|是| B
    E -->|否| F[DH=0, CH+1]
    F --> G{CH < CYLS ?}
    G -->|是| B
    G -->|否| H[读取完毕]
```

- **内层**：遍历一个磁头下的扇区（首次从 S2 开始，之后从 S1 开始）
- **中层**：切换磁头（0 ↔ 1）
- **外层**：切换柱面（0 → `CYLS-1`，共 `CYLS=10` 个）

#### 4. 内存布局图

所有读取的扇区被连续存放在 `ES` 开始的内存中（物理地址 `0x8200` 起）。

```mermaid
block-beta
  columns 1
  block:Physical Memory
    A["0x07C00 ~ 0x07DFF: 引导扇区 (512B)"]
    B["0x07E00 ~ 0x081FF: 未使用 (1KB 间隔)"]
    C["0x08200 ~ 0x0A3FF: C0-H0-S2~S18 (17 扇区)"]
    D["0x0A400 ~ 0x0C7FF: C0-H1-S1~S18 (18 扇区)"]
    E["0x0C800 ~ 0x0EBFF: C1-H0-S1~S18 (18 扇区)"]
    F["..."]
    G["0x34E00 ~ 0x34FFF: C9-H1-S1~S18 (最后 18 扇区)"]
  end
```

**总量计算**：  

- 总扇区数 = `10柱面 × 2磁头 × 18扇区 - 1（引导扇区） = 359`  
- 总字节 = `359 × 512 = 183,808 = 0x2CE00`  
- 结束物理地址 = `0x8200 + 0x2CE00 = 0x35000`，最后一个字节在 `0x34FFF`

#### 5. FAT12 磁盘结构

该引导扇区符合标准的 1.44MB 软盘格式：

| 区域 | 大小 (扇区) | 说明 |
|------|------------|------|
| 引导扇区 | 1 | 位于 C0-H0-S1，BIOS 自动加载 |
| FAT #1 | 9 | 文件分配表 1 |
| FAT #2 | 9 | 文件分配表 2（备份） |
| 根目录区 | 14 | 最多 224 个目录项 |
| 数据区 | 剩余 | 实际文件内容 |

你读取的 359 个扇区已经包含了全部 FAT 表、根目录以及大部分数据区，为后续的文件查找做好了准备。

### 五、常考易错点

1. **段寄存器赋值必须 /16**：`0x8200` → `0x0820`，错写 `0x820` 会导致物理地址偏差 16 倍。
2. **扇区号从 1 开始**，不是 0，CHS 参数中 CL 的范围是 1～18。
3. **首次读取跳过 S1**：因为 S1 已被 BIOS 加载到 `0x7C00`，后续从 S2 开始。
4. **`JB` vs `JBE`**：`JB` 是无符号小于跳转，与 `CMP` 配合时，`CMP DH,2` 后接 `JB readloop` 等价于 `DH < 2`，不会包含 2。
5. **A20 门**：虽然本代码未涉及，但了解段地址 ×16 时产生的回绕现象有助于理解实模式的历史包袱。

### 六、延伸思考

- 如果 `CYLS` 设置过大，会读取到不存在的扇区，BIOS 返回错误，重试 5 次后进入 `error`。
- 当前代码读取了所有扇区到内存，但未解析 FAT 表或根目录，这将是下一阶段实现文件加载的基础。
- 保护模式下段寄存器不再直接存储基址，而是选择子，彻底告别了 ÷16 的手动计算。
